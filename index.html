O Que é GitHub e Para Que Serve

O GitHub é um site que abriga um software de controle de versão de desenvolvimento através do sistema Git. Falando assim, parece complicado né? Mas entender o que é GitHub e para que ele serve é mais simples do que você possa imaginar. Trata-se de um sistema que possibilita trabalhos em colaboração com outros desenvolvedores — estejam eles do seu lado ou em outro continente.

Dúvidas mais frequentes

Abaixo você encontrará as respostas das principais dúvidas relacionadas ao GitHub

Criar o primeiro repositório usando o GitHub Desktop

O GitHub Desktop amplia e simplifica o fluxo de trabalho no GitHub.com com uma interface visual, em vez de comandos de texto na linha de comando.Depois de instalar o GitHub Desktop e entrar no GitHub ou GitHub Enterprise, você pode criar e clonar um repositório de tutorial.
O tutorial apresentará os conceitos básicos de trabalho com o Git e o GitHub, incluindo a instalação de um editor de texto, criando um branch, fazendo um commit, fazendo push para GitHub.com e abrindo um pull request.

Como obtenho acesso ao Github?

Para entrar no GitHub é preciso criar uma conta individual ou para a sua equipe, preenchendo os dados solicitados. Você pode optar pela opção gratuita ou paga. A última conta com algumas vantagens como repositórios privados.

O que é um repositório?

O repositório onde os seus arquivos (códigos, imagens, áudios etc) ficam armazenados, com as opções de depósito do próprio GitHub ou no seu computador

O que é um Branch?

Branch cópia do diretório para desenvolvimento isolado, que não afeta o repositório central ou outros branches. Depois de concluir o seu trabalho, você pode combinar o seu branch isolado com outros branches por meio de uma merge, utilizando o Pull Request.

O que é um Pull Request no Github

Pull Request é quando você informa que irá implantar as mudanças no seu branch ao repositório máster, podendo ser aceito ou negado pelos colaboradores. Além disso, é possível revisar e discutir seu trabalho com os demais.

O que é um Forking?

Forking é a criação de um novo projeto com base em repositório existentes, fazendo as alterações necessárias e armazenando a nova versão como um novo repositório.

Há como criar um repositório no GitHub com um projeto já desenvolvido?

Você pode adicionar um novo origin no seu git. Para isso use o comando git add origin remotenovo url-do-seu-novo-repositorio. Depois basta fazer um commit e um push para o novo repositório: git push -u remotenovo master
